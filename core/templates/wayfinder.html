Wayfinder

An exploration in data visualazation with 360,000 data points



Challenge

Given a set of 360,000 specific 4 digit elevation points, how does one make visual sense of it? Changes in elevation create topographical landscapes so it seemed the best way to attempt to render such data was by creating an image to represent the data set.

The approach was to write a Python script that could take each data point and convert it into a specific value on the color scale (0-255) based on its relationship to the maximum and minimum elevation. When rendered, the result should be a textured topographical looking image in which each pixel represents a single data point.

The Process

  • Clean Data- Take away commas, spaces, and any other characters in the data so it can be used

  • Tools- Pillow is an excellent tool for python for imaging, this is what I used to take each data point and turn it into a single pixel in a picture

  • Height and Depth - Identify max and min data points

  • Colorize - With my max and min numbers defined I could go about turn numbers into color codes using this formula - y - min_height / max_height - min_height * 255

  • Render With every data point set to a color code, Loop through each value and apply color formula , and plot as a single pixel.

The Result

In less than a second I am able to render image into 360,000 pixels (or rather a 600x600 image). Sure, enough this data set looks like a landscape, (yay). It took me a couple tries to get it correctly, but you can see the darkest shades represent the lowest elevations and the brightest representing the highest.
